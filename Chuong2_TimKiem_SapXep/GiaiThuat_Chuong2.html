Tìm Kiếm Tuần Tự

O(N)Tìm kiếm

Duyệt từng phần tử từ đầu đến cuối. Không yêu cầu mảng đã sắp xếp.

Điểm quan sát
  • Mỗi bước kiểm tra một phần tử, không bỏ qua.
  • Tốt nhất O(1) nếu phần tử ở đầu mảng.
  • Xấu nhất O(N) nếu ở cuối hoặc không tìm thấy.
Pseudocode (từ file .cpp)
TimTuyenTinh(a[], n, x): i = 0 while a[i] != x and i < n: i++ if i < n: return i // tìm thấy tại vị trí i return -1 // không tìm thấy

Tìm Kiếm Nhị Phân

O(log₂ N)Tìm kiếm

Chia đôi khoảng tìm kiếm sau mỗi bước. Yêu cầu mảng đã sắp xếp tăng dần.

Điểm quan sát
  • Xanh lam = khoảng [l, r] hiện tại. Cam = mid đang xét.
  • 1000 phần tử → tối đa chỉ 10 bước so sánh.
Pseudocode (từ file .cpp)
TimNhiPhan(a[], n, x): l = 0, r = n-1 while l <= r: m = (l+r) / 2 if a[m] == x: return m else if a[m] > x: r = m-1 // tìm bên trái else: l = m+1 // tìm bên phải return -1

Selection Sort – Chọn Trực Tiếp

O(N²)Sắp xếp

Mỗi lượt tìm phần tử nhỏ nhất từ phần chưa sắp xếp rồi đưa vào vị trí đúng.

Điểm quan sát
  • Đúng N-1 lần hoán vị — ít hơn Bubble Sort.
  • Không ổn định: thứ tự phần tử bằng nhau có thể đổi chỗ.
Pseudocode (từ file .cpp)
SelectionSort(a[], n): for i = 0 to n-2: vt = i for j = i+1 to n-1: if a[j] < a[vt]: vt = j HoanVi(a[i], a[vt])

Insertion Sort – Chèn Trực Tiếp

O(N²)Sắp xếp

Chèn từng phần tử vào đúng vị trí trong đoạn đã sắp xếp — như sắp bài trong tay.

Điểm quan sát
  • Mảng gần sắp xếp → gần O(N), rất tốt trong thực tế.
  • Ổn định: giữ thứ tự phần tử bằng nhau.
Pseudocode (từ file .cpp)
InsertionSort(a[], n): for i = 1 to n-1: pos = i; x = a[i] while pos > 0 and a[pos-1] > x: a[pos] = a[pos-1] // dịch phải pos-- a[pos] = x // chèn vào đúng chỗ

Bubble Sort – Nổi Bọt

O(N²)Sắp xếp

Mỗi lượt duyệt từ phải sang trái, hoán vị hai phần tử kề sai thứ tự. Phần tử lớn "nổi" về cuối.

Điểm quan sát
  • Sau lượt i, phần tử lớn thứ i+1 đã về đúng vị trí cuối mảng.
  • Tối ưu: dừng sớm nếu không có hoán vị nào trong một lượt.
Pseudocode (từ file .cpp)
BubbleSort(a[], n): for i = 0 to n-2: for j = n-1 downto i+1: if a[j] < a[j-1]: HoanVi(a[j], a[j-1])

Quick Sort – Sắp Xếp Nhanh

O(N log N) TBChia để trị

Chọn pivot, phân hoạch trái < pivot < phải, đệ quy hai nửa. Nhanh nhất trong thực tế.

Điểm quan sát
  • Pivot lấy ở giữa a[(l+r)/2] — như code gốc trong file .cpp.
  • Xấu nhất O(N²) khi mảng đã sắp và pivot luôn là min/max.
Pseudocode (từ file .cpp)
QuickSort(a[], l, r): i=l; j=r; x=a[(l+r)/2] // pivot giữa do: while a[i] < x: i++ while a[j] > x: j-- if i <= j: HoanVi(a[i],a[j]); i++; j-- while i <= j if l < j: QuickSort(a,l,j) if i < r: QuickSort(a,i,r)

Heap Sort – Vun Đống

O(N log N)Sắp xếp · Heap

Xây max-heap, liên tục trích phần tử lớn nhất ra cuối. Đảm bảo O(N log N) mọi trường hợp.

Điểm quan sát
  • Giai đoạn 1 — Xây heap từ node n/2-1 về 0: O(N).
  • Giai đoạn 2 — Đổi gốc với cuối rồi Shift lại: O(log N) mỗi bước.
Pseudocode (từ file .cpp)
HeapSort(a[], n): // Giai đoạn 1: xây heap for k = n/2-1 downto 0: Shift(a,k,n-1) // Giai đoạn 2: sắp xếp for r = n-1 downto 1: HoanVi(a[0],a[r]); Shift(a,0,r-1)
